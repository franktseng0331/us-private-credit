**分支**：`feature/data-cleaning`

**清洗输出**：`data/cleaned/deal_positions_clean.csv`  
**计划版本**：v1.0（2026-04-16）

## 一、数据质量问题清单

| 优先级 | 问题 | 影响规模 |
|--------|------|---------|
| P1 | `investment_type` 大量为空或特殊字符 | ~36,000条 |
| P1 | `investment_type` 同义词命名不统一（200+种原始名称） | 全量 |
| P2 | `industry` 字段错误填充（脚注、投资类型、公司名后缀） | ~15,000条 |
| P2 | `borrower_name` 因逗号解析偏移导致实体识别困难 | 部分记录 |
| P3 | 金额单位不一致（部分BDC用千美元，部分用百万美元），且同一BDC可能随年份更换单位 | 影响财务分析 |
| P3 | `cost_basis_usd_mn` / `position_size_usd_mn` 存在负值（需区分真实异常与合理负值） | ~1,300条 |
| P4 | `maturity_raw` 日期格式不统一；股权类资产无到期日需特殊处理 | 全量 |
| P4 | `base_rate` 和 `spread_raw` 字段不完整；PIK 利差未单独提取 | 36.1%缺失 |

## 二、清洗步骤设计（共7步）

### 步骤 1：Investment Type 标准化

**目标**：将 200+ 种原始投资类型映射到约 15 个标准类别

**关键要求**：严格规定匹配优先级（Order of Execution），必须先匹配复合条件，再匹配单一条件，避免带 Delayed Draw / Revolver 的记录被误判为普通 Term Loan。

**标准类别映射规则**（按顺序执行）：

| 优先级 | 匹配关键字 | 标准类别 |
|--------|-----------|---------|
| 1 | `first lien` + `delayed draw` | `First Lien Delayed Draw` |
| 2 | `first lien` + `revolver` | `First Lien Revolver` |
| 3 | `first lien` | `First Lien Term Loan` |
| 4 | `second lien` | `Second Lien Term Loan` |
| 5 | `unitranche` | `Unitranche Loan` |
| 6 | `senior secured` | `Senior Secured Loan` |
| 7 | `subordinated` / `mezzanine` / `mezz` | `Subordinated Debt` |
| 8 | `clo` / `structured finance` / `structured note` | `Structured Finance / CLO` |
| 9 | `common stock` / `common equity` | `Common Equity` |
| 10 | `preferred` | `Preferred Equity` |
| 11 | `warrant` | `Warrant` |
| 12 | `unsecured note` / `senior note` | `Unsecured Note` |
| 13 | `revolver` | `Revolver` |
| 14 | 空白 / 特殊字符 / 无法匹配 | `Unknown` |

**新增字段**：`investment_type_std`；原始 `investment_type` 保留。

---

### 步骤 2：Industry 字段清洗 + GICS 宏观分类映射

**目标**：清除错误填充内容，恢复有效行业标签，并映射到 GICS 宏观大类

**清洗规则**：
1. 以下内容视为无效，替换为空字符串：
   - 脚注格式：`\(\d+\)` 如 `(12) (13)`
   - 是投资类型关键词：loan, lien, secured, equity, warrant, note, stock
   - 公司名后缀：`Inc.`, `LLC`, `Ltd.`, `Corp.`, `LP`, `Co.`
   - 单一特殊字符：`$`, `*`, `-`, `HTML`
2. 清洗后仍为空的记录，通过 `borrower_name_clean`（见步骤7）跨季度回填有效行业值

**GICS 宏观分类映射**（约12个大类）：

| GICS 大类 | 关键行业词 |
|-----------|-----------|
| `Software & Technology` | software, application, technology, internet, SaaS |
| `Healthcare` | health care, pharmaceutical, medical, biotech, life sciences |
| `Business Services` | business services, staffing, consulting, professional services |
| `Financial Services` | financial, insurance, banking, asset management |
| `Industrials` | industrial, manufacturing, aerospace, defense, chemicals |
| `Consumer` | consumer, retail, food, beverage, restaurant |
| `Energy` | energy, oil, gas, power, utilities |
| `Real Estate` | real estate, REIT, property |
| `Media & Telecom` | media, telecom, communication, broadcasting |
| `Education` | education, training |
| `Structured Finance` | structured finance, CLO, ABS |
| `Other / Unknown` | 无法匹配 |

**新增字段**：`industry_clean`（清洗后原始名）、`industry_gics`（宏观大类）；原始 `industry` 保留。

---

### 步骤 3：金额单位统一

**目标**：将所有金额字段统一为百万美元（USD millions）

**关键设计**：判断粒度细化为 **BDC + filing_id（单份财报）**，防范同一BDC随年份更换披露单位。

**判断逻辑**：
```
对每份财报（BDC + filing_id 组合）计算 fair_value_usd_mn 中位数：
  中位数 > 5,000  → 该份财报以千美元为单位 → 三个金额字段 ÷ 1,000
  中位数 < 0.1    → 该份财报以美元为单位 → 三个金额字段 ÷ 1,000,000
  否则             → 保持不变（百万美元）
```

**受影响字段**（就地修正）：`position_size_usd_mn`, `cost_basis_usd_mn`, `fair_value_usd_mn`

---

### 步骤 4：负值异常标记（区分真实异常 vs. 合理负值）

**目标**：区分解析错误导致的异常负值 vs. 财务上合理的负值

**业务背景**：未提取的循环信用（Unfunded Revolvers）和延迟放款定期贷款（Delayed Draw Term Loans）在市场标记恶化时，公允价值可合理为负（表示负债）。只有普通定期贷款（Term Loan）出现负值才绝对是异常。

**标记规则**：

| 条件 | 标记 |
|------|------|
| `fair_value_usd_mn < 0` 且 `investment_type_std` 包含 Revolver 或 Delayed Draw | `is_unfunded_liability = True` |
| `fair_value_usd_mn < 0` 且 `investment_type_std` 为普通 Term Loan 类 | `is_anomaly = True` |
| `cost_basis_usd_mn < 0` 或 `position_size_usd_mn < 0`（任意投资类型） | `is_anomaly = True` |

**新增字段**：`is_anomaly`（boolean）、`is_unfunded_liability`（boolean）

---

### 步骤 5：日期字段标准化

**目标**：将 `maturity_raw` 解析为标准日期，股权类资产允许为空

**支持的格式**：
- `Dec 2027` → `2027-12-01`
- `12/2027` → `2027-12-01`
- `12/31/2027` → `2027-12-31`
- `2027-12-31` → 直接保留
- `N/A` / `Perpetual` / 缺失（股权/权证类资产） → `null`（**不计入解析失败率**）

**新增字段**：`maturity_date`（YYYY-MM-DD 或 null）；`maturity_raw` 保留。

---

### 步骤 6：利率字段补全（含 PIK 拆分）

**目标**：提取数值型利差、标准化基准利率、单独提取 PIK 利差

**背景**：2021-2023 年为 LIBOR → SOFR 过渡期。PIK（实物支付利息）转换是当前私募信贷风险监测的核心指标——将现金利息转为 PIK 通常是信用压力的早期预警信号。格式示例：`"S+550 (1% PIK)"`。

**新增/更新字段**：

| 字段 | 说明 | 示例 |
|------|------|------|
| `base_rate`（更新） | 统一映射为 SOFR / LIBOR / PRIME / Fixed | `SOFR` |
| `spread_bps`（新增） | 总利差数值，整数，基点 | `550` |
| `pik_spread_bps`（新增） | PIK 利差数值，整数，基点（无PIK则为0） | `100` |

**提取逻辑**：
- `spread_bps`：正则匹配 `+\d+\.?\d*\s*(bps|%)` 后换算为基点
- `pik_spread_bps`：正则匹配 `(\d+\.?\d*)\s*%?\s*PIK` 后换算为基点
- 交叉验证：若 `pik_spread_bps > 0` 则检查 `is_pik` 是否为 True，不一致时修正 `is_pik`

---

### 步骤 7：借款人名称标准化（Entity Resolution）

**目标**：创建清洗后的借款人名称，用于跨季度追踪同一借款企业的估值轨迹

**业务价值**：同一家底层公司可能在不同财报中被写为 "Acme Corp"、"Acme Corporation"、"Acme, LLC"。标准化是实现步骤2跨季度回填行业的前提，也能正确追踪同一公司的 Mark-to-Market 轨迹。

**规则**（按顺序执行）：
1. 转为全大写
2. 去除常见企业后缀：`INC`, `LLC`, `LTD`, `CORP`, `LP`, `CO`, `HOLDINGS`, `GROUP`
3. 去除所有标点符号（逗号、句号、连字符、括号）
4. 去除多余空格，trim

**新增字段**：`borrower_name_clean`；原始 `borrower_name` 保留。
